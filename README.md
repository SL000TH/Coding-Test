1. 알고리즘 성능평가
   1) 복잡도: 낮을 수록 Good!
      - 시간 복잡도: 특정한 크기의 입력에 대하여 알고리즘의 수행 시간 분석
      - 공간 복잡도: 특정한 크기의 입력에 대하여 알고리즘의 메모리 사용량 분석
     
   2) 복잡도 표기 방법
      - 빅오 표기법: 가장 빠르게 증가하는 '항'만 고려 ex) N^3 + N^2 + N -> N^3 
        
2. 알고리즘 설계 Tip!!!
   1) 코딩 테스트 문제에서 시간제한은 통상 1~5초 가량
   2) 문제에서 가장 먼저 확인 해야 하는 내용은 시간제한(수행시간 요구사항) 
   3) 알고리즘 문제 해결 과정
      - 지문읽기 및 컴퓨터적 사고 - 요구사항(복잡도)분석 - 문제 해결을 위한 아이디어 찾기 - 소스코드 설계 및 코딩
